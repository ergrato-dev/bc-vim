# Ebooks Gratuitos — Semana 10

Planificación de la configuración final, publicación de dotfiles, documentación y presentación, flujo de demo y comunidad Neovim — los recursos de lectura para cerrar el bootcamp con un proyecto final publicable.

---

## Imprescindibles esta semana

### Practical Vim (2nd Edition) — Repaso Completo
- **Autor**: Drew Neil
- **URL**: https://pragprog.com/titles/dnvim2/
- **Capítulos recomendados**: Repásalo de principio a fin como checklist de dominio — cada capítulo corresponde a una capa de la configuración de `01-planificacion-configuracion.md`
- **Por qué**: Es el libro que acompañó todo el bootcamp; esta semana no aprendes contenido nuevo del libro, lo usas para verificar que tu configuración final cubre lo esencial de cada capítulo.

### Learn Vim (the Smart Way)
- **Autor**: Igor Irianto (iggredible)
- **URL**: https://github.com/iggredible/Learn-Vim
- **Por qué**: Su estructura modular (un tema por capítulo) es un buen espejo para organizar el README de tu propio repositorio de dotfiles, tema de `02-publicacion-dotfiles.md`.

### Neovim Lua Guide
- **URL**: https://neovim.io/doc/user/lua-guide.html
- **Por qué**: Referencia oficial para revisar que tu `init.lua` y módulos en `lua/` siguen las convenciones recomendadas antes de publicarlos — la base técnica de `01-planificacion-configuracion.md`.

---

## Complementario

### Learn Vimscript the Hard Way
- **Autor**: Steve Losh
- **URL**: https://learnvimscriptthehardway.stevelosh.com/
- **Por qué**: Si tu configuración final todavía mezcla Vimscript legacy con Lua, este libro te ayuda a decidir qué migrar y qué documentar como está en el README, relevante para `02-publicacion-dotfiles.md`.

---

> Esta semana no hay lecturas nuevas por tema técnico — el trabajo es de síntesis. Usa Practical Vim como checklist de lo aprendido y Learn Vim como referencia de cómo documentarlo bien en tu propio repositorio.
