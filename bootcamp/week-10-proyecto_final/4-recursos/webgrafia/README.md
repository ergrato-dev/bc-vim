# Webgrafía — Semana 10

Referencias web para planificar, publicar, documentar y presentar tu configuración final de Neovim, y para seguir aprendiendo después del bootcamp.

---

## Referencia rápida

### Neovim Lua Guide (neovim.io)
- **URL**: https://neovim.io/doc/user/lua-guide.html
- **Esta semana**: Referencia oficial de la API Lua para verificar que tu `init.lua` y módulos siguen las convenciones recomendadas — base de `01-planificacion-configuracion.md`

### `:help` (vimhelp.org)
- **URL**: https://vimhelp.org/
- **Esta semana**: Sigue siendo el primer lugar para verificar cualquier opción u comando antes de incluirlo en tu configuración final o documentarlo en el README

---

## Artículos y guías recomendadas

### Learn Vim (the Smart Way) — GitHub
- **URL**: https://github.com/iggredible/Learn-Vim
- **Por qué**: Su README modular es un buen modelo de organización y tono para el README de tu propio repositorio de dotfiles, tema de `02-publicacion-dotfiles.md`.

### kickstart.nvim
- **URL**: https://github.com/nvim-lua/kickstart.nvim
- **Por qué**: Configuración mínima y muy comentada — referencia de cómo documentar cada línea de un `init.lua` monolítico, útil al escribir tu propia documentación en `03-documentacion-y-presentacion.md`.

### LazyVim
- **URL**: https://github.com/LazyVim/LazyVim
- **Por qué**: Distribución modular con lazy.nvim — referencia de estructura de carpetas (`lua/plugins/`, `lua/config/`) para comparar contra la tuya, tema de `01-planificacion-configuracion.md`.

### Dotfyle
- **URL**: https://dotfyle.com
- **Por qué**: Plataforma para indexar y compartir configuraciones de Neovim — el lugar recomendado para publicar la tuya una vez esté en GitHub, tema de `02-publicacion-dotfiles.md`.
